About This Portal

Architecture, data source, GAS script, and deployment guide

🗂️ Data Architecture
All data lives in a Google Sheet (12 tabs). HTML fetches each tab as Published CSV on every load — no backend, no API keys, no server.

Flow: Google Sheet → Publish as CSV → HTML fetches on load → Renders live cards + KPIs + carousels
🔐 Security Model
• Password gate: SHA-256 client-side hash — prevents casual access
• All data stays in Google Sheets — Sheet owner controls access
• No credentials in HTML — Sheets published read-only CSV
• GitHub Pages: Static hosting only — zero server exposure
• GAS script: Runs inside Google — no external exposure
⚡ GAS Script — One Trigger Only
The only GAS script needed is GAS_Timestamp_Script.js — it auto-updates the last_updated cell in the Meta sheet whenever any cell is edited.

Install: Extensions → Apps Script → Paste script → Save → Triggers → onAnyEdit → On edit
🚀 Deployment Steps
1. Import XLSX template to Google Sheets
2. File → Publish to Web → each sheet → CSV
3. Copy Sheet ID from URL
4. Replace YOUR_SHEET_ID in HTML
5. Push HTML to GitHub repo
6. Settings → Pages → Enable → Done
7. Access via rhchheda.github.io/…
